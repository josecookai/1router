## Source

## Acceptance Criteria
1. `POST /api/keys` returns plaintext key only at creation time and persists only hash/prefix metadata.
2. `GET /api/keys` lists key metadata without exposing plaintext.
3. Response payloads match `openapi/mvp.yaml` list/create schemas for API keys.
4. Tests verify plaintext key is not stored in repository snapshots and list responses.

## Scope
- In scope: in-memory hash-only API key storage, create/list routes, tests, OpenAPI contract updates.
- Out of scope: DB persistence, revoke/rotate endpoints, auth enforcement, scoped keys.

## Test Plan
- Unit: key generation/hash helpers and store persistence shape.
- Integration: create key then list keys, invalid create payload.
- CI: `make test`, `make coverage`, `make ci`.
