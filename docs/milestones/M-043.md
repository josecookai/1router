## Acceptance Criteria
1. 为 payment webhook 事件 ID 提供带 TTL 的幂等存储（过期后允许重新处理）。
2. 重复事件（同 event id + 同 payload）返回 replay 指示且不触发重复副作用。
3. 同 event id + 不同 payload 返回确定性冲突错误 envelope。
4. 覆盖 first delivery / duplicate replay / mismatch guard / TTL expiry 测试。
5. `make test`, `make coverage`, `make ci` 全绿且覆盖率达标。

## Scope
- In scope: in-memory TTL idempotency store、payment webhook replay/mismatch 行为、测试与文档。
- Out of scope: 持久化存储、跨进程共享幂等、provider 特定签名校验。
