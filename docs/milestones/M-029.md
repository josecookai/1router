# M-029: Route Decision Trace Explain Endpoint

## Acceptance Criteria
1. Add `/v1/routing/decision/explain` endpoint returning route decision factors (`preset` weights, selected provider/model, ranked candidates).
2. Response must not include secrets (bearer token, key hashes, provider credentials).
3. Add route tests covering happy path, malformed request, and no-secret guarantee.

## Scope
- In scope: explainability API for routing decisions and tests.
- Out of scope: persistence, streaming explain, and provider live telemetry.

## Validation
- `make test`
- `make coverage`
- `make ci`
