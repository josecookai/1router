## Acceptance Criteria
1. SQL-first migration framework is added (`scripts/db/migrate.sh`) and local apply command is documented in `README.md`.
2. `db/migrations/0001_mvp_schema.sql` creates the MVP billing/control-plane schema on a clean PostgreSQL instance.
3. Core tables exist after migration: `orgs`, `api_keys`, `providers`, `models`, `usage_events_raw` (+ `usage_events_raw_default` partition), `invoices`, `audit_logs`.
4. A Postgres migration smoke test is available in local/CI automation (`tests/postgres_migration_smoke_test.sh`) and re-run behavior is verified via migration bookkeeping.

## Scope
- In scope: SQL migration runner, initial schema migration, smoke verification, developer docs.
- Out of scope: ORM integration (Prisma/Drizzle), seed data, repository/query layer implementation.
