## Acceptance Criteria
1. Add inbound payment webhook endpoint with idempotency by provider event id.
2. Duplicate event id with same payload returns deterministic replay response.
3. Duplicate event id with different payload returns 409 conflict envelope.
4. Add tests for first delivery, replay, conflict, and invalid request.
5. `make test`, `make coverage`, `make ci` pass with gates.

- Out of scope: persistent dedup store, provider-specific signature verification, queue fanout.
