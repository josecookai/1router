# Milestone M-005

## Source
- Issue: #6
- Area: area/infra

## Acceptance Criteria
1. Every request gets a `request_id` exposed via `x-request-id` response header and included in error envelopes.
2. Unhandled and validation errors return a shared JSON error envelope with stable `code/message/request_id` fields.
3. Sensitive header values (`Authorization`, API/provider keys) are redacted by helper logic and logger redaction paths are configured.
4. Tests cover `request_id` generation, error envelope formatting, validation formatting, and redaction behavior.

## Scope
- In scope: Fastify middleware/error-handler baseline and tests.
- Out of scope: auth, rate limiting, secrets management, streaming route special-casing.

## Test Plan
- Unit: redaction helper tests.
- Integration: `/healthz` request id, `/boom` error envelope, `/validate` schema validation envelope.
- CI: `make test`, `make coverage`, `make ci`.
