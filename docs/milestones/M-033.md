# Milestone M-033

## Source
- Issue: #65
- Area: area/ui

## Acceptance Criteria
1. `GET /api/models` supports optional `provider` and `status` filters.
2. Response shape stays OpenAPI-compatible and includes capability flags for each model.
3. Invalid query values return shared 400 error envelope.
4. Tests cover happy path, filtered query, and invalid query.
5. `make test`, `make coverage`, `make ci` pass with coverage gates.

## Scope
- In scope: control-plane model list/filter behavior, schema alignment, tests.
- Out of scope: UI redesign, model pricing dashboards, persistence changes.
